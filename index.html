一、SM Agent 定位与工作流程
从需求分析结果到要素级设计，逐层细化
SM Agent 目标：将需求分析阶段识别出的改动点（如“新增出资方”、“支持新还款模式”），结合渠道接入要素模型（出资比例、还款模式、支付渠道、减免分配、融担费参数等），落实到具体系统模块的要素字段，输出结构化的设计变更清单。

二、渠道接入要素模型（示例）
基于梳理的核心要素，按系统归属分类

出资与还款

服务清分
• 出资比例（如阳光消金100%出资）
• 还款模式（NF, NM, FS）
• 分期期数（3/6/9/12）
• 还款试算方式（海尔/资方）

支付与渠道

资金账务
• 支付渠道（宝付/京东支付等）
• 渠道号转换规则
• 签约收单方式
• 是否指定扣款渠道

减免与费用

资产核算 服务清分
• 减免类型（36%减免/优惠券）
• 减免分配（资方承担/海尔摊销）
• 融担费参数（NA/FG/PA）
• 提前还款手续费

代偿与买断

服务清分
• 代偿规则（D+4代偿，累三买断）
• 买断后还款资金流（线上/线下）
• 是否入贷后
• 买断前/后还款流程

核算与批扣

资金账务 资产核算
• 还款试算（海尔/资方）
• 批扣发起方（海尔/不发起）
• 拆分扣支持
• 记账方式（验资/不验资）

通知与通用

服务清分
• 短信路由
• 通用记账配置
• 融担公司信息
